## **4. Step-by-Step Execution**

### **4.1 Register Route (`/api/auth/register`)**

* Validates email with `validator.isEmail()`
* Validates strong passwords with `validator.isStrongPassword()`
* Hashes password using `bcrypt`
* Saves user to MongoDB

```bash
curl -X POST http://localhost:5000/api/auth/register \
-H "Content-Type: application/json" \
-d '{"email":"test@gmail.com","password":"StrongPass123!"}'
```

**Expected Output:**

```json
{"message":"User registered"}
```

---

### **4.2 Login Route (`/api/auth/login`)**

* Verifies email exists
* Compares password hash
* Returns JWT token valid for 1 hour

```bash
curl -X POST http://localhost:5000/api/auth/login \
-H "Content-Type: application/json" \
-d '{"email":"test@gmail.com","password":"StrongPass123!"}'
```

**Expected Output:**

```json
{"token":"<jwt_token_here>"}
```

---

### **4.3 Protected Dashboard Route (`/api/dashboard`)**

* Uses `authMiddleware` to verify JWT
* Only accessible with valid token

```bash
curl -X GET http://localhost:5000/api/dashboard \
-H "Authorization: Bearer <jwt_token_here>"
```

**Expected Output:**

```json
{"message":"Welcome to your dashboard, user 699fd3738dbe60a1442d0bff!"}
```

---

## **5. Security Measures Implemented**

1. **Input Validation:** Prevents invalid email & weak password submissions
2. **Password Hashing:** Securely stores passwords with bcrypt
3. **JWT Authentication:** Ensures only logged-in users can access protected routes
4. **Protected Routes:** Middleware verifies token before access
5. **HTTP Header Security:** Helmet reduces common web vulnerabilities
